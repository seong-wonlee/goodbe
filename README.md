# 현대오토에버 - 철강 MES(최종 탈)

- **자기소개서**
    
    *1.**현대오토에버의 해당 직무에 지원한 이유와 앞으로 현대오토에버에서 키워 나갈 커리어 계획을 작성해주시기 바랍니다.***
    
     학부 4학년 때, 철강회사에서의 생산관리 인턴 경험이 해당 직무에 지원하는 데 큰 영향을 미쳤습니다. 당시 회사에서는 생산 현황을 실시간으로 모니터링하고 재고를 관리하는 웹 시스템이 구축 되어있어, 제철소 생산관리 업무의 효율과 생산성을 향상하고 있었습니다. 이를 보며 생산 현장에서 소프트웨어 기술을 활용하는 것의 중요성을 몸소 깨닫게 되었고, 이러한 시스템을 직접 개발하여 생산성 향상에 기여하는 소프트웨어 개발자 직무에 큰 매력을 느꼈습니다. 
    
     현대오토에버에 입사한 뒤, 먼저 제조 시스템 개발 분야에서의 전문성을 더욱 향상하고자 합니다. Java 활용 능력과 Oracle DB 관리 기술, 이를 활용하는 생산 도메인에 대한 이해를 높임으로써 제조 현장을 효율적으로 운영할 수 있는 MES를 구축하겠습니다. 또한, 계속해서 기술 동향과 트렌드를 주시하고 학습하여 현대오토에버가 MES 기술의 선두 주자로서 더욱 발전할 수 있도록 만들겠습니다. 이를 통해 현대오토에버와 저의 제조 시스템 개발자 커리어의 동반성장을 이뤄내겠습니다.

     산업공학과 학부 시절 Java 프로그래밍을 처음 접하고 큰 흥미를 느꼈기에, 졸업 후 삼성 청년 SW 아카데미(SSAFY)에 입과 하여 Java 코딩 트랙을 이수하였습니다. SSAFY 학기 말에는 배운 내용을 총망라하는 웹 페이지 구현 프로젝트를 수행했습니다. 2명의 백엔드, 프론트엔드 페어로 구성된 팀에서 Spring 프레임워크와 MySQL DB를 설계 및 관리하는 백엔드를 담당했습니다. 
    
     웹 프로젝트의 주제는 ‘사용자 맞춤 운동 추천 사이트’였습니다. 운동 취향 테스트를 통해 적합한 운동을 추천해 주고, youtube API, 구글 및 카카오 소셜 로그인을 활용해 회원을 관리하고 운동 관련 정보를 제공해 주는 사이트를 기획했습니다. Web Browser - Controller – Service – DAO – Mapper - DateBase로 이어지는 구조를 처음부터 제작해 보며, Spring 프레임워크와 MVC 패턴에 대한 이해를 높일 수 있었습니다. 또한 MyBatis 기반의 mapper 작성으로 Spring을 MySQL DB와 연결하는 과정에서, 초기 DB 설계 시의 문제점을 발견하여 DB를 재설계하는 경험을 했습니다. 
    
     이 경험을 통해 웹 프로젝트 진행 시, 사전에 충분한 논의를 통해 DB 설계를 진행해야 함을 깨닫게 되었습니다. 서버를 Vue.js로 구현된 프론트엔드와 연결하기 위해 REST API를 활용하였습니다. 이를 통해 클라이언트와 서버를 분리함으로써, 코드의 모듈화와 API 명세의 가독성을 높일 수 있어 협업이 용이했습니다. 
    
     하지만 연결 과정에서 Spring의 Controller와 Service 구현부의 문제점들이 드러났습니다. 프론트엔드를 담당한 페어와 지속적으로 소통하며 더 효율적인 방식으로 코드를 개선하였고, 이 경험을 통해 개발 프로젝트에서 팀원과의 소통의 중요성을 다시 한번 깨닫게 되었습니다. 웹 프로그래밍 경험 외에도, 저는 철강 회사의 생산관리 부서에서 인턴을 한 경험이 있습니다. 당시 부서에는 냉연 공장 생산 계획을 최적화하는 개선과제가 있었습니다. 과제 수행을 위해 제철소의 생산 관리 프로세스 전반 대한 이해가 필요했기에, 퇴근 후에도 인턴 동기들과 자발적으로 남아 철강 공정 스터디를 진행했습니다. 프로세스에 대한 이해를 바탕으로 한 달 동안 팀원들과 과제에 몰두한 결과, 해당 개선과제로 발표회에서 전체 38개 참가팀 중 2위로 은상을 수상하는 쾌거를 이룰 수 있었습니다.

    ### MES란?
    
    MES는 **제조 실행 시스템(Manufacturing Execution System)**의 약자로, 생산관리를 지원하기 위해 사용되는 소프트웨어 시스템입니다. MES는 생산 현장에서 발생하는 데이터를 수집하고 분석하여 생산 프로세스를 모니터링하고 관리하는 역할을 수행합니다.
    
    MES는 생산 공정의 실시간 모니터링, 자동화, 자동 데이터 수집, 자동 보고 등 다양한 기능을 제공하여 생산 현장의 효율성과 생산성을 향상시킵니다. 일반적으로 MES는 다음과 같은 주요 기능을 포함합니다.
    
    1. 생산 계획 관리: MES는 생산 계획을 생성하고 관리하는 기능을 제공합니다. 생산 계획은 주문 수량, 생산 라인의 가용성, 재고 상태 등을 고려하여 최적의 생산 계획을 수립합니다. 또한, 생산 계획에 따라 생산 자원 및 재료의 예약과 할당을 수행합니다.
    2. 생산 현장 모니터링: MES는 생산 현장에서 발생하는 데이터를 실시간으로 모니터링합니다. 예를 들어, 생산 라인의 운영 상태, 장비의 가동률, 제품의 생산 진행 상황 등을 모니터링하여 생산 현황을 실시간으로 파악할 수 있습니다. 이를 통해 생산 현장에서 발생하는 문제를 조기에 감지하고 대응할 수 있습니다.
    3. 생산 자원 관리: MES는 생산에 필요한 자원을 관리합니다. 이는 장비, 공구, 부품 등을 포함할 수 있습니다. MES는 생산 자원의 가용성을 파악하고 예약, 할당, 추적하는 기능을 제공하여 생산 계획의 원활한 진행을 도와줍니다.
    4. 품질 관리: MES는 생산품의 품질을 관리하는 기능을 제공합니다. 생산 공정에서 발생하는 데이터를 수집하고 분석하여 제품의 품질을 평가하고, 품질 이상이 발생한 경우 조치를 취할 수 있도록 도와줍니다. 또한, MES는 품질 검사와 테스트 결과를 기록하고 추적하여 품질 보증에 필요한 정보를 제공합니다.
    5. 작업지시 관리: MES는 생산 작업을 지시하고 관리하는 기능을 제공합니다. 작업자에게 작업 지시를 할당하고, 작업 진행 상황을 모니터링하며, 작업 결과를 기록합니다. 이를 통해 생산 작업의 효율성과 정확성을 높일 수 있습니다.
    6. 데이터 분석 및 보고: MES는 생산 데이터를 수집하고 분석하여 관련 보고서를 생성합니다. 예를 들어, 생산량, 가동률, 품질 지표 등의 데이터를 분석하여 생산 성과를 평가하고 관련된 보고서를 생성합니다. 이를 통해 경영자나 생산 관리자는 생산 현황을 신속하게 파악하고 의사 결정을 내릴 수 있습니다.
    
    MES는 생산 공정을 효율적으로 관리하기 위한 핵심 시스템으로서, 생산 현장의 실시간 모니터링과 데이터 분석을 통해 생산성을 향상시키고 생산 비용을 절감하는 데 도움을 줍니다.
    
    ### ERP와 MES의 관계
    
    ERP(Enterprise Resource Planning)는 생산, 판매, 자재, 인사, 회계 등 기업의 전반적인 업무 프로세스를 하나의 체계로 통합/재구축하여 정보를 서로 공유하고 신속한 업무처리를 도와 주는 전사적 자원관리 시스템이다. 
    
    반면 MES는 생산 관리 시스템(Manufacturing Execution System)으로 기업의 생산 현장에서 작업 일정, 작업 지시, 품질 관리, 작업 실적 집계 등 제반 활동을 지원하기 위한 관리 시스템을 말한다. 
    
    ERP와 MES는 서로 다른 영역을 관리하지만, 효과적인 상호작용을 통해 기업의 운영을 최적화할 수 있다. 상호작용 하는 방식은 다음과 같다. 
    
    1. 데이터 통합: ERP와 MES는 데이터를 공유하고 통합하는 방식으로 상호작용합니다. MES 시스템은 생산 현장에서 발생하는 데이터를 수집하고 ERP 시스템과 공유합니다. 이를 통해 ERP 시스템은 생산 관련 데이터를 활용하여 예산 관리, 재고 관리, 생산 계획 등의 기능을 수행할 수 있습니다.
    2. 생산 계획과 실행: ERP 시스템은 고객 주문 및 수요 예측 데이터를 기반으로 생산 계획을 수립합니다. 이때 MES 시스템은 생산 가능성과 생산 라인의 가용성 등을 고려하여 실제 생산 계획을 수립하고 실행합니다. MES 시스템은 생산 현장의 운영 상태를 모니터링하고 ERP 시스템에 생산 진행 상황을 실시간으로 전달하여 생산 계획의 조정을 지원합니다.
    3. 자원 관리: ERP 시스템은 기업의 자원 관리를 담당하며, MES 시스템은 생산에 필요한 자원(장비, 부품 등)의 예약, 할당, 추적을 수행합니다. MES 시스템은 ERP 시스템에 생산 자원의 가용성 정보를 제공하여 생산 계획의 자원 관리를 지원합니다.
    4. 품질 관리: MES 시스템은 생산품의 품질을 관리하는 기능을 제공하며, ERP 시스템은 품질 관련 데이터를 수집하여 품질 보증과 관련된 기능을 수행합니다. MES 시스템은 생산 공정에서 발생하는 품질 데이터를 ERP 시스템에 제공하고, ERP 시스템은 이를 기반으로 품질 관리와 관련된 결정을 내립니다.
    
    위의 방식으로 ERP와 MES가 상호작용하면서 기업은 생산 공정을 효율적으로 관리하고 기업 전체적인 운영 성과를 향상시킬 수 있다. ERP와 MES의 상호작용은 기업의 운영에 있어서 중요한 역할을 수행하며, 생산성과 품질을 향상시키는 데 기여한다. 
    
    ### MES 개발자
    
    - 주요 업무
        - 통합정보시스템(ERP/MES)개발 및 유지보수
        - 생산관리 모듈에 대한 장애 조치, 개선 작업, 시스템의 운영 계획 수립 + 신규 서비스 개발
    - MES 개발 직무의 장단점
        - 장점
            - 현업의 도메인 지식과 전산 지식 등 다양한 업무 지식 습득, 시스템의 큰 그림을 볼 수 있는 안목이 생김
            - MES 관련 프로젝트들이 많으므로 차후 이직이나 프리랜서 활동에 유리
            - 일반 IT 직무에 비해 신기술의 압박을 덜 받는다. 대신 시스템의 안정성을 최우선에 두고 필요할 땐 응용 가능한 기술을 사용하여 실리를 추구하는 업무 특성
        - 단점
            - 24시간 가동되는 라인 특성상 휴일이나 퇴근 시간 후 시스템 장애에 긴급 대응해야 할 수도 있음.
            - 개발 업무, 장애 처리등의 업무를 동시에 처리해야 함.
            - 차세대같은걸 잘 안하기 때문에 스파게티 소스가 심각한 경우 다수(*그래도 하다보면 레거시 분석 및 소스 리딩 능력이 향상…)*
    - MES 개발자가 갖춰야 할 역량
        - 제조 시스템에 대한 이해
            - 담당하는 현장에 최적화 된 시스템을 구축해야 함
            - 전체 프로세스(=업무 흐름도)를 파악하는 것이 우선적
            - 제품 발주부터 출고까지의 전사적인 프로세스 파악
            - 프로세스를 잘 파악하고 발생한 이슈에 대해 얼마나 빠르고 효율적으로 대응 할 수 있는지
        - 커뮤니케이션
            - 현업 담당자와의 협업 과정에서 요구 사항을 명확하게 파악하고 내용을 전달하는 능력
            - 고객사, 현업 담당자와 협의점을 찾는 의사소통 능력
            - 프로그램을 혼자 개발하는 것이 아니기 때문에 팀원과의 소통, 명확한 의사전달 필요
        - 기본적인 프로그래밍 능력/센스
            - 프로그래밍 언어는 시스템의 특성과 환경에 맞추어 다양하게 바뀔 수 있기 때문에, 언어에 영향 받지 않고 빠르게 습득할 수 있는 능력 필요
            - 신입에게는 다른 사람의 소스를 파악할 수 있는 능력이 필요
        - 책임감
            - 시스템을 운영하다 보면 장애는 시와 때를 가리지 않고 발생함. 이러한 장애를 어떻게 조치하냐 에 따라 재발을 방지할 수도 있고, 그렇지 않을 수도 있다.
        - 현장 작업자의 니즈를 정확하게 파악
            - **고객의 관점에서 문제를 바라보고 생각할 수 있는 능력**
        - 개인 역량 향상을 위한 노력 예시
            - *신규 서비스가 추가될 수 있기 때문에 다양한 프로그래밍 언어와 신기술을 꾸준히 습득해보고 현 시스템에 적용 가능한지 보아야 합니다. 이에 저는 Python을 활용한 자체 시스템 개발을 KPI 목표로 잡고 자기 계발을 진행 중입니다. 또한, PMP(Project Management Professional) 자격증 취득도 준비하고 있습니다.*
        - 화려한 스포트라이트가 없어도 내가 만든 시스템이 누군가에게 도움이 된다는 것으로 보람을 느낄 수 있는 긍정적 마인드
    - MES 개발 업무에서 중요한 점
        - 시스템의 안정성
            - 생산과정에서 초 단위로 트랜잭션이 발생하고, 장애 발생시 생산라인이 중지될 수 있는 Mission Critical*한 특징이 있기 때문에 무엇보다 안정성이 중요하다.
                
                (*Mission Critical: 정상적으로 작동하지 않으면 업무 수행 전체에 치명적인 영향을 초래하는 분야)
                
        - 데이터의 정확도
            - MES 프로그램 특성상, 재고나 생산 정보, 설비 등의 현물 데이터를 관리하므로 일반 웹 개발에 비해 데이터의 높은 정확도가 요구된다.
        - 개발 속도

        - 개발/생산 등의 부서들과 유기적인 협업

    - 상사와 대화가 통하지 않는다면 어떻게 할 것인지?
        
        먼저 상사분이 어떤 스타일인지 분석하고, 거기에 맞추면서 살갑게 다가가려고 최대한 노력해 볼 것 같습니다. 그래도 소통이 잘 되지 않는다면 회사안에는 최소한 한 명이라도 그 분과 소통이 잘 되는 분이 계실 것이기 때문에, 그 분을 찾아가서 제가 어려워 하는 분과 소통하는 방법에 대해 조언을 들어볼 것 같습니다.
